# Reflection

## What I Built
A client-side AI spend audit tool that helps startups find savings on AI tool subscriptions by comparing their current spend against official pricing data.

## What Went Well
- Audit engine logic came together faster than expected
- Tailwind CSS made the UI fast to build
- localStorage persistence worked cleanly without a backend

## What Was Hard
- Getting the audit recommendations to feel "smart" and defensible rather than generic
- Balancing simplicity with enough logic to be genuinely useful
- Time constraint of 3 days forced prioritization

## What I Would Do Differently
- Use a proper backend (Supabase) for email capture and shareable URLs
- Add TypeScript for better maintainability
- Add more tools (Perplexity, Grammarly, Jasper, etc.)
- Add a usage frequency input (daily vs occasional) for smarter recommendations

## What I Learned
- Real pricing data changes frequently — a live pricing API would be better than hardcoded data
- Entrepreneurial products need to balance polish with speed of shipping
- Documentation is as important as the code itself
